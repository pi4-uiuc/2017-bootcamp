Version control makes it possible to have ‘unlimited undo’ and histories of your documents (and some smaller datasets)

Having everything on a website like GitHub (or Bitbucket or GitLab) makes it easy to share with other people and collaborate. If you keep all of your important programs and files somewhere in the cloud, like on box.com, dropbox, github, it make it easier to use heterogeneous environments.

Version Control SWC Git Novice 1-6

Getting Started

Introduce yourself

git config --global user.name "Firstname Lastname"
git config --global user.email you@yourdomain.example.com

If you want your contributions to be recognized:

mkdir myproject
git init
git status
cat "my project" > README.md
git status

commiting changes

You need to tell git which files to line up for the commit

git add README.md
git status
git commit -m "added README as first commit"
git status

branching

git checkout -b first-script

This creates a new branch called your ‘first script’. Now lets add something to it.

echo "!#/bin/bash \n echo 'hello world'" > myscript.sh
  • What is \n?
  • What does $ do?

Now commit the changes:

git add myscript.sh
git commit -m 'added hello world script'

Merging

git status
git log
git checkout master
git status
git merge first-script
git status

Review

git log --graph
